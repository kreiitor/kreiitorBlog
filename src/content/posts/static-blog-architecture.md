如果博客目标只是发几篇随笔，任何静态站工具都够用。但如果目标是持续沉淀论文、项目、实验和博客，站点结构就需要更明确地服务内容增长。

## 为什么选择 Astro

我这次选择 Astro，主要因为它兼顾了几个需求：

- 页面本身是静态输出，适合 GitHub Pages
- 内容集合对 Markdown 管理很友好
- 页面路由清晰，后续扩展项目页和标签页成本低

## 第一版应该优先做什么

第一版最重要的不是高级交互，而是把内容流搭通：

1. 首页能同时展示博客和项目。
2. 文章页有标签、阅读时间和评论位。
3. 标签页、归档页和搜索页能覆盖常见浏览路径。

## 对 GitHub Pages 友好的能力

为了保证部署简单，我倾向于优先使用纯静态方案：

- 搜索使用 Pagefind
- 评论使用 Giscus
- RSS 和 sitemap 直接在构建阶段生成

## 设计上避免“模板味”

即使是技术博客，也不一定要长成标准文档站。更适合个人学术主页的首页，应该同时表达：

- 我在研究什么
- 最近写了什么
- 有哪些项目值得看

这也是为什么首页需要更强的首屏视觉，但内部内容页仍然保持技术博客的清晰结构。
